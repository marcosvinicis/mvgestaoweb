# Protocolo CORE MV™ – SEO, Performance e Conversão

Este documento define as regras inegociáveis de SEO Técnico, Performance, Estrutura Semântica, Autoridade e Conversão.
Nenhum código ou copy pode ser gerado ignorando estas diretrizes.

---

## 1. Core Web Vitals (Obrigatório)

- LCP < 2.5s
- CLS < 0.1 (todas as imagens e vídeos devem conter width e height explícitos)
- INP < 200ms
- JavaScript sempre com defer
- CSS Mobile-First
- Imagens obrigatoriamente em formato .webp
- Evitar bibliotecas desnecessárias
- CSS crítico pode ser inline na Hero

---

## 2. Estrutura Semântica

- Apenas um H1 por página
- Hierarquia obrigatória: H1 → H2 → H3
- Uso correto de <header>, <main>, <section>, <footer>
- Links com âncoras semânticas (proibido "clique aqui")
- Alt text descritivo (sem spam)

---

## 3. Helpful Content + E-E-A-T

- Responder à intenção de busca na primeira dobra
- Linguagem natural (proibido keyword stuffing)
- Palavra-chave principal presente:
  - No H1
  - Na URL
  - Na meta description
- Prova social obrigatória
- Contato visível (WhatsApp fixo)

---

## 4. Dados Estruturados

- JSON-LD obrigatório validável
- Institucional: Organization ou ProfessionalService
- Negócio Local: LocalBusiness
- Blog: Article
- FAQ: FAQPage
- Navegação: BreadcrumbList

---

## 5. Arquitetura de Conversão

Ordem obrigatória da página:

1. Intenção de busca (Hero)
2. Problema
3. Solução
4. Prova Social
5. FAQ
6. CTA Final

WhatsApp é o CTA principal.
Evitar múltiplos CTAs concorrentes.

---

## 6. Rodapé Estratégico

O footer deve conter:

- Nome da empresa
- Endereço completo
- Cidade e região de atuação
- Telefone com DDD
- Horário de atendimento
- Links internos estratégicos
- Política de Privacidade
- Termos de Uso
- Créditos:

"© Ano Atual – Nome da Empresa. Todos os direitos reservados."
"Estrutura Digital desenvolvida por MV Gestão Web – Marketing Digital em Maringá"

---

## 7. Regra Suprema

Se qualquer decisão violar este protocolo,
ela deve ser corrigida antes do commit.
